# 总部后台权限

## 角色权限定义（数据库 admin_roles）

### 超级管理员（role-super-admin）
- 权限：`['*']`
- 说明：全部页面可见

### 总管理员（role-admin）
- 权限：`["operators:list","players:read","marketing:read","finance:read","dashboard:read","dashboard:list"]`
- 说明：可看运营商管理、营销、财务、数据看板

### 运营管理员（role-ops-admin）
- 权限：`["operators:read","operators:list","dashboard:read"]`
- 说明：可看运营商管理、数据看板

### 财务管理员（role-finance-admin）
- 权限：`["finance:read","finance:withdraw","finance:history"]`
- 说明：仅财务相关

# 硬编码权限覆盖

## auth.ts 中的硬编码规则
- **总部超管 `admin`**：`auth.ts` 的 admin-login 接口中 `if (user.username === 'admin') permissions = ['*']`
- **运营商超管 `13944444444`**：`auth.ts` 的 operator-login 接口中返回 `permissions: ['*']`

## 权限优先级
1. 硬编码 `permissions: ['*']` > 数据库角色权限
2. 手动修改数据库 `admin_roles.permissions` 对于 `admin` 和 `13944444444` 无效

## 前端过滤逻辑
- 菜单项无 `perms`（如个人中心）→ 所有人可见
- `permissions.includes('*')` → 全部菜单可见
- 用户 `permissions` 不包含菜单所需的 `perms` → 隐藏

# 用户体系归属（完整数据库映射）

> ⚠️ **重点规则：** 修改用户相关逻辑前，必须参考此表确认角色对应的数据库表。

## 数据库表归属

| 角色 | 数据库表 | 登录方式 | 关键字段 | 页面入口 |
|------|---------|---------|---------|---------|
| **总部超管** | `admin_users` | 用户名/手机号 | `role_id=role-super-admin` | `/admin/login` |
| **总部总管理员** | `admin_users` | 手机号 | `role_id=role-admin`, `operator_id=null` | `/admin/login` |
| **总部运营管理员** | `admin_users` | 手机号 | `role_id=role-ops-admin`, `operator_id=null` | `/admin/login` |
| **总部财务管理员** | `admin_users` | 手机号 | `role_id=role-finance-admin`, `operator_id=null` | `/admin/login` |
| **运营商超管** | `operators` | `operator_username`+密码 | `operators` 表本身 | `/operator/login` |
| **运营商总管理员** | `admin_users` | 手机号 | `role_id=op_admin`, `operator_id` 关联运营商 | `/operator/login` |
| **运营商财务** | `admin_users` | 手机号 | `role_id=op_finance`, `operator_id` 关联运营商 | `/operator/login` |
| **裁判员** | `referees` + `users` | **手机号+密码**（H5网页） | `referees.phone` 登录，`referees.user_id` 关联 `users` | `/referee/login` |
| **玩家** | `users` | 微信登录（小程序） | `openid` 唯一标识，`subscribe_venue_id` 关联场馆 | 玩家小程序 |

## 角色互斥规则

### ❌ 不可共存（同一手机号）
- **管理角色互斥**：同一个人不能同时是两个管理员角色
  - 不能同时是总部超管 + 运营商超管
  - 不能同时是总部总管理员 + 运营商财务
  - 不能同时是同一个运营商的超管 + 总管理员
  - 不能同时是总部管理员 + 运营商管理员（任意组合）

### ✅ 可以共存（同一手机号）
- **管理员 + 玩家**：可以同时是总部管理员 + 小程序玩家
- **管理员 + 裁判员**：可以同时是运营商超管 + H5裁判员
- **玩家 + 裁判员**：可以同时是玩家 + H5裁判员

## 运营商超管 vs 角色成员（重点）

| 对比项 | 运营商超管 | 运营商角色成员 |
|-------|-----------|--------------|
| 所在表 | `operators` | `admin_users` |
| 创建方式 | 总部后台→运营商管理→添加运营商 | 运营商后台→角色与成员管理→创建成员 |
| 显示在角色成员页面 | ❌ 不显示 | ✅ 显示 |
| 可否被删除 | ❌ 需要先禁用运营商 | ✅ 可单独删除 |
| 权限硬编码 | `auth.ts` 返回 `permissions: ['*']` | 按 `admin_roles.permissions` 解析 |
| 登录入口 | 运营商后台 | 运营商后台 |

## 裁判员登录流程

```
POST /api/v1/auth/login (role不指定或为空)
  │
  ├── 查 referees 表（r.phone 匹配）
  │     ├── 有记录 → bcrypt验证密码
  │     │              ├── 成功 → 返回token → 进入H5裁判端
  │     │              └── 失败 → 返回401
  │     └── 无记录 → Mock回退（临时逻辑）
  │
  └── 说明：裁判端是H5网页，不是小程序
      裁判账号由运营商在后台创建（referees表）
      密码存储在 referees.password（bcrypt）
      关联 users 表记录（user_id）
```

## 玩家归属规则

- `users.subscribe_venue_id` = `venues.id`
- `venues.operator_id` = `operators.id`
- 有 `subscribe_venue_id` → 运营商旗下玩家
- 无 `subscribe_venue_id` → 总部直属玩家
- 总部后台 `scope=operator` → 查 `subscribe_venue_id IS NOT NULL`
- 总部后台 `scope=direct` → 查 `subscribe_venue_id IS NULL`
