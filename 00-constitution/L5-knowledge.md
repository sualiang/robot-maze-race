## 📖 什么是 L5 知识层

L5 知识层是体系的**经验沉淀层**。

- L1 宪法告诉你"什么不能做"（红线）
- L2 岗位告诉你"你该做什么"（职责）
- L3 规范告诉你"什么是对的"（标准）
- L4 技能告诉你"具体怎么做"（步骤）
- **L5 知识告诉你"别人踩过什么坑、有什么好经验"（经验）**

L5 知识层的内容来自于实际项目的沉淀，是最接地气、最有实战价值的内容。

## 🎯 核心价值

### 对个人
- 快速学习，站在别人的肩膀上
- 避免重复踩坑，少走弯路
- 提升解决问题的效率

### 对团队
- 知识共享，打破信息孤岛
- 新人快速上手，降低培训成本
- 经验传承，人员流动不影响
- 持续改进，团队能力越来越强

### 对体系
- 自进化的基础 — 经验积累多了，可以反过来优化 L1-L4 层的规则
- 验证规则的效果 — 从实践中检验规则是否合理
- 让体系越来越完善，越来越好用

## 📝 沉淀原则

### 什么值得沉淀
1. **花了很长时间才解决的问题** — 你花了一天才搞定，别人可能也会花一天，沉淀下来大家都省时间
2. **容易踩的坑** — 你踩了，别人也可能踩，提前告诉大家
3. **好的做法、最佳实践** — 摸索出了好方法，分享给大家
4. **可复用的代码、方案** — 不用每次都从零开始
5. **有启发的经验教训** — 虽然不是技术问题，但对做事有启发

### 什么不值得沉淀
1. **太简单、太常识的东西** — 大家都知道，没必要沉淀
2. **一次性的、特定场景的问题** — 只有你这个项目会遇到，别人不会遇到
3. **质量太低、写不清楚的** — 别人看了也看不懂，不如不沉淀
4. **敏感信息、商业机密** — 绝对不能沉淀

## 🔄 沉淀与进化的闭环

L5 知识层不是孤立的，它和上面几层形成一个闭环：

```
做项目 → 积累经验 → 沉淀到 L5 知识层
    ↑                            ↓
    └── 优化规则 ← 提炼总结 ← 归纳共性
        (L1-L4)
```

1. **做项目** — 在实际项目中积累经验
2. **沉淀** — 把经验沉淀到 L5 知识层
3. **归纳** — 从大量经验中归纳出共性的规律
4. **优化** — 把这些规律提炼成规范、流程、模板，更新到 L3/L4 层
5. **复用** — 下一个项目用优化后的规则，做得更好

这样，体系就会像滚雪球一样，越用越强，越用越完善。

## 👤 各角色使用要求

### 所有角色
1. **遇到问题先查知识库** — 不要上来就问，先自己找找
2. **解决了问题及时沉淀** — 不要转头就忘，花两分钟记下来
3. **有好的经验主动分享** — 不要藏着掖着，分享出来大家一起进步

### 技术开发总监
1. 整体负责知识层的建设和维护
2. 推动沉淀机制落地
3. 定期回顾知识层内容，推动向上提炼

### 各领域负责人
1. 负责对应领域知识的质量
2. 评审新沉淀的内容
3. 定期整理和优化对应领域的知识
