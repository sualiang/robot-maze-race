## 📖 什么是 L4 技能层

L4 技能层是体系的**操作手册层**。

- L1 宪法告诉你"什么不能做"（红线）
- L2 岗位告诉你"你该做什么"（职责）
- L3 规范告诉你"什么是对的"（标准）
- **L4 技能告诉你"具体怎么做"（步骤）**
- L5 知识告诉你"别人踩过什么坑"（经验）

## 🎯 为什么需要 L4 技能层

1. **提高效率** — 不用每次都从零开始，直接复用成熟的流程和模板
2. **保证质量** — 按标准流程走，减少出错概率
3. **便于协作** — 大家都用同一套流程，沟通成本低
4. **知识沉淀** — 把个人经验变成团队能力
5. **新人友好** — 新人来了照着做就行，不用到处问

## 📜 使用规则

### 强制要求
1. **所有项目必须遵循标准化工作流**，不得随意跳过关键环节
2. **有现成解决方案的，必须优先复用**，禁止重复造轮子
3. **文档输出必须使用标准模板**，保证格式统一、信息完整
4. **工作流的关键节点必须留痕**（评审记录、审批记录等）

### 维护职责
- **技术开发总监** — 技能层总负责人，统筹规划和质量把控
- **系统架构师** — 技术类工作流和解决方案的维护
- **产品总负责人** — 产品类工作流和模板的维护
- **项目管理专员** — 项目管理类工作流和模板的维护
- **所有角色** — 使用过程中发现问题及时反馈，有好的经验及时沉淀

### 沉淀原则
1. **做完一个项目，沉淀一批资产** — 项目结项时必须完成知识沉淀
2. **成熟了再入库** — 经过验证的方案才能进入技能层
3. **持续优化** — 不是放进去就不管了，要根据实际使用情况持续迭代
4. **分类清晰** — 放到正确的分类下，便于查找和复用
