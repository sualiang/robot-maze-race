## 📋 经验列表

### 1. Session 压缩禁止开启
- **背景：** 默认的 auto-compaction 会自动压缩对话记录，导致 Agent "失忆"
- **问题：** 压缩后无法搜索之前聊过的内容，重要决策和上下文丢失
- **教训：** 这不是功能bug而是设计决策，但对我们来说压缩的坏处远大于好处
- **改进建议：** openclaw.json 中设置 `compaction.enabled = false`，每次升级后检查该配置是否被重置
- **相关项目：** OpenClaw 体系配置
- **时间：** 2026-06-21

### 2. gateway 冷重启才能清内存缓存
- **背景：** 持续运行的 OpenClaw gateway 进程累积内存缓存（API 连接、模型路由等）
- **问题：** `gateway restart`（SIGHUP）仅热重载不清缓存，老进程长时间不重启会缓存失效
- **教训：** 必须 `gateway stop` + `gateway start` 才能彻底清内存缓存
- **改进建议：** 故障时优先用 stop+start，不要 restart
- **相关项目：** OpenClaw 运维
- **时间：** 2026-06-12

### 3. WebChat 多账户切换需清除 Cookie
- **背景：** 配置了多个 OpenClaw Agent（小D + 豆包），通过 /dm 切换
- **问题：** 切换频道后 UI 输入框和回答身份仍显示前一个 Agent
- **教训：** WebChat 不原生支持多账户切换，身份标签存在缓存
- **改进建议：** 清除浏览器 Cookie 或从新入口登录才能刷新显示
- **相关项目：** OpenClaw WebChat 配置
- **时间：** 2026-06-26

### 4. sessions.visibility 在 V2 runtime 硬编码无法配置
- **背景：** 需要在 openclaw.json 中控制 sessions 显示模式
- **问题：** `tools.sessions.visibility` 和 `sessions.visibility` 均已从 V2 schema 移除
- **教训：** 不是配置没生效，是配置项根本不存在
- **改进建议：** 不要花时间排查不存在的配置项；影响不大的 feature 等官方更新
- **相关项目：** OpenClaw 配置
- **时间：** 2026-06-26

### 5. 三端不一致导致开发冲突
- **背景：** 凌晨在本地大量修改代码未提交，然后直接开发新功能
- **问题：** 本地代码（改版后）、GitHub（原始版）、服务器（原始版）三端不一致，新功能基于旧代码开发导致冲突
- **教训：** 开工前必须做 git status 和三端对比
- **改进建议：** 新任务开发前执行版本对齐检查 SOP
- **相关项目：** 匹配赛小程序
- **时间：** 2026-06-18

### 6. 产品文档内容可能不全面
- **背景：** 豆包写的产品文档里写了裁判端/商家端要做小程序
- **问题：** 实际上裁判端、运营商端、商家端都走 H5 网页，不搞小程序
- **教训：** 产品文档可能不全面，特别是涉及端的划分时
- **改进建议：** 看到产品文档里的端划分描述，先跟 Allen 确认再执行
- **相关项目：** 匹配赛项目
- **时间：** 2026-06-18

### 7. 规则文件引用旧编号导致混淆
- **背景：** V3.5 重组编号后，部分规则文件正文仍引用旧编号（如"上报系统架构师"、"通知项目管理专员"）
- **问题：** 使用时产生混淆，不清楚该找谁
- **教训：** 编号重组后必须检查所有文件中的引用是否同步更新
- **改进建议：** 版本升级时做一次全文引用审计
- **相关项目：** SASDT V3.5 体系升级
- **时间：** 2026-06-26
