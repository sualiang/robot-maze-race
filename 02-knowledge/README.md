## 📖 什么是 L5 知识层

L5 知识层是体系的"经验宝库"，所有项目积累的经验、教训、最佳实践都沉淀在这里。

- L1 宪法告诉你"什么不能做"
- L2 岗位告诉你"你该做什么"
- L3 规范告诉你"什么是对的"
- L4 技能告诉你"具体怎么做"
- **L5 知识告诉你"别人踩过什么坑、有什么好经验"**

## 🎯 为什么需要知识层

1. **避免重复踩坑** — 别人踩过的坑，你就不要再踩了
2. **提升开发效率** — 有现成的最佳实践和代码片段，直接拿来用
3. **知识传承** — 新人来了能快速上手，老人走了知识不流失
4. **持续改进** — 经验积累多了，可以反过来优化体系规则
5. **团队成长** — 大家互相学习，共同进步

## 📝 怎么贡献

### 什么时候贡献
1. **项目结项时** — 系统整理项目中的经验教训，这是主要的贡献时机
2. **日常随时** — 遇到有价值的东西，随时记下来，后面再整理
3. **解决了疑难问题后** — 特别是花了很长时间才解决的 Bug，一定要沉淀
4. **发现了好方法后** — 自己摸索出了好的做法，分享给大家

### 贡献要求
1. **真实** — 必须是真实发生的，不是编的
2. **有价值** — 别人看了能学到东西，不要凑数
3. **清晰** — 写清楚背景、过程、结果、原因
4. **分类准确** — 放到正确的文件和分类下
5. **格式规范** — 按照各文件的格式要求来写

### 贡献流程
1. 写好内容
2. 提交给对应领域的负责人评审
3. 评审通过后，正式加入知识库
4. 通知大家有新的知识沉淀

## 🔍 怎么使用

### 遇到问题时
1. 先到知识层找找有没有类似的问题
2. 找到的话，参考现成的解决方案
3. 解决后，如果有新的心得，记得更新回去

### 做方案时
1. 先看看有没有相关的最佳实践
2. 参考别人的经验，少走弯路
3. 有更好的方案，记得沉淀回去

### 新项目启动时
1. 先看看经验教训和 Bug 案例
2. 提前规避常见的坑
3. 把别人的经验变成自己的

## 📜 维护规则

### 日常维护
1. 各文件的维护人负责对应内容的质量和更新
2. 定期整理，删除过时的内容，补充新的内容
3. 分类要清晰，便于查找

### 质量把控
1. 新内容必须经过评审才能入库
2. 定期审查已有内容，质量差的要清理或优化
3. 重复的内容要合并，不要重复

### 安全注意
1. 禁止沉淀敏感信息、密钥、密码、内部地址
2. 禁止沉淀商业机密、用户隐私数据
3. 涉及第三方的内容要注意版权
