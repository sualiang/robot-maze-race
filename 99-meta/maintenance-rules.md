# MCP 层维护规则

> **MCP 标准化上下文层的维护规范**
>
> 所有角色都必须遵守

---

## 🎯 维护目标

1. 确保 MCP 层内容准确、及时、完整
2. 确保所有角色都能方便地获取所需上下文
3. 确保知识和经验能够有效沉淀和复用
4. 确保 MCP 层本身的结构清晰、易于维护

---

## 👥 维护职责

### 技术总调度（小D）— 总负责人
- MCP 层整体架构设计与优化
- 目录结构调整与规范制定
- 重要内容的审核与归档
- 定期检查 MCP 层的完整性和准确性
- 协调各角色的维护工作

### 各岗位角色 — 内容贡献者
- 及时更新自己专业领域的知识和上下文
- 确保自己负责的内容准确、完整
- 发现问题及时反馈或修复
- 项目结束后及时沉淀知识和经验

### 独立审计守护进程 (com.sasdt.discipline-inspector) — 外部监督者
- 审计 MCP 层的执行情况
- 检查是否有角色绕过 MCP 层
- 检查 MCP 层内容是否符合体系规则
- 发现违规及时告警

### 项目管理专员 — 进度与项目信息维护
- 维护活跃项目列表
- 维护全局待办事项
- 项目结项后组织知识沉淀

---

## 📝 更新规则

### 什么时候需要更新 MCP 层
1. 有新的重要决策时
2. 项目状态有重大变化时
3. 有新的知识或经验沉淀时
4. 发现错误或过时信息时
5. 体系规则有变更时

### 更新流程
1. 找到对应的文件和位置
2. 修改内容，确保准确、清晰
3. 注明更新时间和更新人
4. 重要更新通知相关角色
5. 记录到更新日志中（可选，重大变更才记）

### 更新原则
1. **及时** — 有新信息尽快更新，不要拖延
2. **准确** — 确保内容正确，不要写错
3. **完整** — 该说的要说清楚，不要遗漏
4. **简洁** — 不要啰嗦，重点突出
5. **分类** — 放到正确的位置，不要乱放

---

## 🔒 安全规则

1. **禁止**在 MCP 层存放任何密钥、密码、敏感配置
2. **禁止**在 MCP 层存放用户隐私数据、敏感业务数据
3. **禁止**在 MCP 层存放内部地址、账号等敏感信息
4. 不确定能不能放的，先问技术总调度，不要随便放

---

## 🧹 定期清理

1. **每周** — 检查一次活跃项目和待办事项
2. **每月** — 清理一次过时信息和重复内容
3. **每季度** — 整体回顾一次 MCP 层结构，优化调整
4. **项目结项后** — 整理沉淀该项目的知识和经验

---

## ⚠️ 违规处理

1. 不按要求加载 MCP 层上下文 — 一般违规，警告
2. 重要信息不及时同步到 MCP 层 — 一般违规，警告
3. 在 MCP 层存放敏感信息 — 严重违规，触发熔断
4. 故意篡改或删除 MCP 层重要内容 — 严重违规，触发熔断
5. 多次违规不改正 — 升级处理，上报人类
