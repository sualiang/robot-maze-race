## V3.6 架构标准 — 以「安博机器人」项目为通用模板

### 1. 项目目录标准
```
/opt/{项目名}/          ← 正式生产区
/opt/{项目名}-test/     ← 测试区（如有）
```

### 2. 技术栈标准（安博机器人基线）
```
后端: Node.js 22 + Express (TypeScript)
前端管理后台: Vue 3 + Vite + Nginx 容器化
小程序: 微信原生小程序
数据库: MySQL 8.0 (Docker)
缓存: Redis 7 (Docker)
ORM: Prisma
容器: Docker Compose 编排
```

### 3. Docker Compose 服务标准
| 服务 | 镜像 | 容器名规则 | 端口映射 |
|------|------|-----------|---------|
| MySQL | mysql:8.0 | {project}-mysql | 127.0.0.1:3306（正式） / 127.0.0.1:3316（测试） |
| Redis | redis:7-alpine | {project}-redis | 127.0.0.1:6379（正式） / 127.0.0.1:6380（测试） |
| Backend | 项目构建 | {project}-backend | 127.0.0.1:3001（正式） / 127.0.0.1:3011（测试） |
| Admin | Vue+Nginx | {project}-admin | 0.0.0.0:8080（正式） / 127.0.0.1:9090（测试） |

### 4. 数据库命名标准
- 正式库: `{项目名}`（如 robotrent）
- 测试库: `{项目名}_test`（如 robotrent_test）
- 用户: app（统一）
- 端口: 正式 3306，测试 3316

### 5. 环境隔离
- 正式区 Docker Compose project name: `robotrent`
- 测试区 Docker Compose project name: `robotrent-test`
- 操作前必须确认目录是否带 `-test` 后缀，杜绝混淆

### 6. 生产安全规则
- 数据库端口仅绑定 127.0.0.1（禁止外网暴露）
- Backend 端口仅绑定 127.0.0.1（通过 nginx 反代对外）
- Admin 对外端口限制来源 IP（或通过 CDN/WAF）
- 所有服务配置健康检查
- 日志限制：max-size 10m, max-file 3-5
