# 🔴 安全操作检查清单 V2.0

**更新于 2026-06-26，对齐全局宪法 V3.5 密钥与数据安全铁律（第六、七章）**

**每次执行高危操作前，必须逐项自检。任一不通过 → 停下操作，先问 Allen。违背后果见最后。**

---

## 第一关：判断是否需要检查

问自己：**"这条操作会修改或删除数据吗？"**

包括但不限于：
- 删除文件/数据库/目录 → ✅ 需要检查
- 写入/编辑文件 → ✅ 需要检查
- 执行 SQL（UPDATE/DELETE/DROP/ALTER/TRUNCATE/清除数据） → ✅ 需要检查
- 修改配置 → ✅ 需要检查
- 部署到正式服务器 → ✅ 需要检查
- 代码 push/merge/deploy → ✅ 需要检查
- 生产环境 SSH 登录 → ✅ 需要检查
- 启动/重启服务 → ❌ 通常不需要（除非影响数据）

如果 ✅，进入第二关。

---

## 第二关：对照宪法铁律

**打开 `rules/00-全局宪法级执行规则V3.5.md`，逐条检查以下问题：**

### 🔴 基础检查（每次必须查）

| # | 检查项 | 依据 | 通过 |
|---|--------|------|------|
| 1 | 这条操作涉及**数据库写入**（DELETE/DROP/ALTER/TRUNCATE/重建/批量UPDATE）？→ **必须先问 Allen**，操作前确认备份可用 | 宪法 §6.2 | ☐ |
| 2 | 这条操作涉及**正式代码仓库**（git push/merge/deploy）？→ **必须先问 Allen** | 宪法 §6.3 | ☐ |
| 3 | 这条操作涉及**正式服务器 SSH 登录**？→ **必须先问 Allen**，声明执行任务和有效期限 | 宪法 §6.4 | ☐ |
| 4 | 这条操作涉及**权限/角色/菜单改动**？→ 动手前先读 `权限对照表.md` 和 `已修复Bug清单.md` | MEMORY 铁律 | ☐ |
| 5 | 这条操作由**小D（02岗）**执行？→ **小D不拥有最高权限**，涉及数据/代码仓库/SSH 需 Allen 授权 | 宪法 §7.1 | ☐ |
| 6 | 这条操作涉及**密钥/敏感信息**传递到日志、配置、代码中？→ 必须确保不出现明文 | 宪法 §6.1 | ☐ |

## 第三关：执行

**如果任一检查项不通过 → 停下操作，先问 Allen。**

**如果全部通过 → 可以执行，但执行后必须记录：**
1. 做了什么操作（写入 memory/Y-m-d.md）
2. 是否影响了已有数据
3. 审计日志是否已采集
