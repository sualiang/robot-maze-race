# V3.5密钥安全自动化加固需求
## 1. 本次安全改造总目标
解决vault密钥文件权限无自动巡检、废弃密钥无加密备份、日志truncate触发审计daemon误限流三大安全漏洞，补齐V3.5密钥安全基线自动化能力。
## 2. 分工（V3.5 12岗）
- 统筹负责人：02 技术开发总监（小D）
- 业务脚本开发：03 后端开发
 1）`vault-permission-check.js` 密钥权限定时巡检告警脚本
 2）`secret-backup-encrypt.js` 密钥删除前离线AES加密备份脚本
 配套加固：密钥写入自动chmod 600、全局密钥日志脱敏、gitignore屏蔽密钥文件、全操作审计IPC埋点、OpenClaw定时任务注册
- 系统运维配置落地：07 安全合规&DevOps专员
 部署审计daemon专属logrotate轮转配置，禁止truncate截断日志，改用mv分割，调整进程stdout输出，消除误触发riskLevel=2限流问题
## 3. 验收标准（供01产品做测试校验）
1. 密钥文件权限非600/目录非700，1小时内自动上报审计告警；
2. 删除任意密钥前自动生成加密离线备份，无明文泄露；
3. 日志轮转切割全程不触发资产异常限流，正常core密钥读取access+ACK双向闭环正常；
4. 所有密钥读写、修改、备份、删除操作永久留存审计日志；
5. 密钥完整明文不会打印至任何业务日志。
