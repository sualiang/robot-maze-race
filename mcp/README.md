# MCP 标准化上下文层 — 总入口

> **Model Context Protocol** — SASDT V3.3 体系核心机制
>
> 所有 AI 角色的共享上下文中心，确保全体系在同一个认知坐标系下工作。

---

## 🎯 什么是 MCP 层

MCP（Model Context Protocol）标准化上下文层是 SASDT V3.3 体系的核心基础设施，它：

1. **统一认知基准** — 所有 agent 共享同一套体系规则、项目信息、知识沉淀
2. **消除信息孤岛** — 避免每个 agent 各自为政，信息不同步
3. **保障权责一致** — 所有角色都基于同一套规则和上下文开展工作
4. **支撑自进化** — 知识和经验沉淀到 MCP 层，全体系共享受益

---

## 📂 MCP 层结构

### ⭐ global_context.md — 全局上下文基准（V3.6 更新）

宪法 3.1 强制文件，**所有推理实例启动时强制加载**。

- `global_context.md` — 全角色共享的唯一通用基准（跨项目缺陷分级、测试规范、数据结构框架）。**不包含任何单一项目的业务逻辑。**

---

### 00-constitution/ — 体系规则索引
L1-L5 层规则的统一索引与导航，所有角色必须加载。

- `L1-constitution.md` — 全局宪法要点与全文位置
- `L2-roles.md` — 3个岗位概览与权责索引（V3.6精简）
- `L3-specs.md` — 专业规范概览与索引
- `L4-skills.md` — L4 技能层索引（工作流、解决方案、模板）
- `L5-knowledge.md` — L5 知识层索引（最佳实践、经验教训、Bug案例）

### 01-project/ — 项目上下文【隔离规范】
**每个项目独立创建目录，存放该项目专属业务上下文，不污染通用底座。**

`_template/` 为新建项目的模板目录，新项目参考此模板创建自己的上下文目录。

**核心原则：公共底座 vs 业务数据严格分离**
- `rules/`、`specs/`、`skills/`、`mcp/00-constitution/`、`global_context.md` — 纯通用基础设施，**禁止写入任何单一项目的业务规则**
- `mcp/01-project/[项目名]/` — 存放该项目专属的需求、业务逻辑、验收标准、决策记录
- 业务规则不当隔离的 → 独立审计Daemon有权退修
  - `overview.md` — 项目概述
  - `architecture.md` — 技术架构
  - `progress.md` — 当前进度
  - `decisions.md` — 重要决策记录

### 02-knowledge/ — 公共知识库
跨项目可复用的知识沉淀，全体系共享。

- `best-practices.md` — 最佳实践
- `bug-cases.md` — Bug 案例库
- `code-snippets.md` — 常用代码片段
- `lessons-learned.md` — 经验教训

### 03-shared-memory/ — 共享记忆
全局共享的实时信息，所有角色都可以读写。

- `active-projects.md` — 当前活跃项目列表
- `important-decisions.md` — 全局重要决策记录
- `todo.md` — 全局待办事项

### 99-meta/ — MCP 元数据
MCP 层自身的维护规则与日志。

- `update-log.md` — 更新日志
- `maintenance-rules.md` — 维护规则

## 📜 使用规则

### 🔴 强制要求
1. **所有 agent 开始工作前，必须先读取本文件，了解 MCP 层结构**
2. **必须加载 `00-constitution/` 下的体系规则**，这是最低要求
3. **参与具体项目时，必须加载对应项目的 `01-project/` 上下文**
4. **重要决策、知识沉淀、经验教训，必须同步回 MCP 层对应目录**
5. **禁止绕过 MCP 层直接开展工作**（宪法红线）

### 🟡 维护职责
- **技术开发总监（小D）** — MCP 层总负责人，负责整体维护、结构优化、运行保障
- **各岗位角色** — 负责更新自己相关领域的知识和上下文
- **独立审计守护进程** — 负责审计 MCP 层的执行情况和合规性（位于OpenClaw外部）

### 🟢 更新原则
1. 实时更新 — 有新的重要信息立即同步
2. 分类准确 — 放到对应的目录下，不要乱放
3. 留痕可追溯 — 重要更新记录时间和责任人
4. 不重复 — 已经有的内容不要重复写，更新原文件即可

## 🚀 快速开始

### 新 agent 接入三步曲
1. **读我** — 先读本文件，了解 MCP 层结构
2. **读规则** — 读取 `00-constitution/` 下的体系规则
3. **读项目** — 如果参与项目，读取对应项目的上下文

### 有新信息要沉淀？
1. 判断属于哪一类（规则/项目/知识/记忆）
2. 找到对应的目录和文件
3. 更新内容，注明时间和来源
4. 重要更新同步给技术开发总监

---

## ⚠️ 注意事项

- MCP 层是**共享资源**，修改前请确认不会影响其他角色
- 敏感信息（密钥、密码、隐私数据）**禁止**放入 MCP 层
- 不确定放哪里的内容，先放到 `03-shared-memory/`，由技术开发总监整理归档
- MCP 层的内容是**全体系可见**的，不要写只适合某个角色看的内容
