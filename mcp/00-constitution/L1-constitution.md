## 📋 核心要点速览

### 六大核心原则（V3.6 生效）
1. **人类终审原则** — 所有重大决策、核心变更、红线事项最终裁决权属于人类；产品/测试判定下沉至内置豆包API后，仅重大上线保留人工复核
2. **权责分离原则** — 裁判与运动员绝对分离，禁止角色越权操作、跨级指挥
3. **最小权限原则** — 仅授予完成工作必需的最小权限，权限变更必须走审批流程
4. **全程留痕原则** — 所有操作必须可追溯、可审计，禁止隐瞒、篡改操作记录
5. **成本可控原则** — 严格按分级标准调用模型算力，禁止擅自升级高成本模型
6. **自进化沉淀原则** — 所有项目必须沉淀可复用资产与经验，持续迭代体系规则与能力

### 规则效力层级（从高到低）
1. **L1 宪法层** — `rules/00-全局宪法级执行规则V3.6.md`
2. **L2 岗位层** — `rules/` 目录下 01~02（精简 3 岗 V3.6 执行规则）
3. **L3 规范层** — 技术编码、安全、设计、接口等专业领域规范
4. **L4 技能层** — 标准化工作流、可复用解决方案、通用工具模板
5. **L5 知识层** — 项目经验、Bug案例、最佳实践、踩坑记录

**下位规则不得违反上位规则。**
